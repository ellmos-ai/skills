# Privat-Mail-Writer - Signaturen

Diese Datei enthält neutrale Regeln für Grußformeln und Signaturen. Keine echten Namen, Adressen, Telefonnummern oder privaten Signaturen in dieser Skill-Datei speichern.

## Grundregel

Die Signatur aus dem Kontaktprofil oder aus den letzten gesendeten Mails ableiten. Wenn keine Daten verfügbar sind, eine neutrale Platzhalter-Signatur verwenden.

## Signatur-Modi

| Modus | Verwendung | Grußformel |
|---|---|---|
| `formal` | Behörden, Institutionen, offizielle Termine, unbekannte Kontakte | `Mit freundlichen Grüßen` |
| `halbformal` | bekannte Fachkontakte, Dienstleister, distanzierte private Kontakte | `Viele Grüße` oder `Freundliche Grüße` |
| `privat` | vertraute private Kontakte | aus Kontaktprofil ableiten |
| `kurz` | sehr knappe organisatorische Antworten | kürzeste im Profil belegte Grußformel |

## Ableitung aus Kontaktprofil

Beim Profilaufbau auf diese Signale achten:

- Wird geduzt oder gesiezt?
- Beginnen gesendete Mails mit "Hallo", "Liebe/r", "Sehr geehrte/r" oder ohne Anrede?
- Welche Grußformel nutzt der Kontoinhaber gegenüber diesem Kontakt?
- Wird der Vorname, volle Name oder keine Signatur verwendet?
- Sind Mails sehr knapp oder enthalten sie warme Übergänge?

## Platzhalter

```text
Mit freundlichen Grüßen
[Name]
```

```text
Viele Grüße
[Name]
```

```text
Liebe Grüße
[Name]
```

## Nicht tun

- keine echten Signaturen in diese Datei eintragen
- keine fremde Identität annehmen
- keine KI-Signatur anhängen, außer der User verlangt sie
- keine Adresse, Telefonnummer oder Rolle ergänzen, wenn sie nicht sicher bekannt ist
